# Phase 2: Trang Projects & Blog (Hardcode UI, Mock Data)

## Mục tiêu

Xây dựng UI cho trang Projects và Blog với dữ liệu mock/hardcode, chưa cần tích hợp API. Đảm bảo giao diện hoàn chỉnh, có thể chuyển đổi giữa các trang, hỗ trợ responsive và tối ưu trải nghiệm người dùng.

## Danh sách task chi tiết

### 1. Trang Projects

- [ ] Thiết kế layout tổng thể cho trang Projects (danh sách dự án)
- [ ] Tạo component ProjectCard hiển thị thông tin dự án (mock data)
- [ ] Hiển thị danh sách các ProjectCard (mock data)
- [ ] Thêm phân trang hoặc load-more (nếu cần, dùng mock data)
- [ ] Thiết kế trang chi tiết dự án (Project Detail Page)
- [ ] Điều hướng từ danh sách sang chi tiết dự án
- [ ] Responsive cho toàn bộ trang Projects
- [ ] Tối ưu UI/UX (hover, hiệu ứng, trạng thái empty, loading giả lập)

### 2. Trang Blog

- [ ] Thiết kế layout tổng thể cho trang Blog (danh sách bài viết)
- [ ] Tạo component BlogPostCard hiển thị thông tin bài viết (mock data)
- [ ] Hiển thị danh sách BlogPostCard (mock data)
- [ ] Thêm phân trang hoặc load-more (nếu cần, dùng mock data)
- [ ] Thiết kế trang chi tiết bài viết (Blog Detail Page, render markdown giả lập)
- [ ] Điều hướng từ danh sách sang chi tiết bài viết
- [ ] Responsive cho toàn bộ trang Blog
- [ ] Tối ưu UI/UX (hover, hiệu ứng, trạng thái empty, loading giả lập)

### 3. Chung cho cả Projects & Blog

- [ ] Tối ưu SEO cơ bản (title, meta tag, heading hợp lý)
- [ ] Thêm breadcrumb hoặc navigation nếu cần
- [ ] Viết comment, chuẩn hóa code, cập nhật tài liệu nếu có thay đổi

---

> Lưu ý: Tất cả dữ liệu sử dụng mock/hardcode, tập trung hoàn thiện UI/UX. Sau khi hoàn thành phase này sẽ tiến hành tích hợp API và xử lý dữ liệu động ở phase tiếp theo.
